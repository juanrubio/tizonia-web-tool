{% trans "About Tizonia OpenMAX IL" %}

{% blocktrans %}Tizonia OpenMAX IL is an experimental implementation for Linux of the OpenMAX IL 1.2 provisional specification from The Khronos Group.{% endblocktrans %}

{% blocktrans %}Tizonia OpenMAX IL consists of a number of resources:{% endblocktrans %}

  • A fairly complete implementation of a base OpenMAX IL component library (written in C)
  • An IL Core implementation (written in C)
  • An OS abstraction and utility library (written in C), including POSIX threading and synchronization primitives, FIFO and priority queues, dynamic array, config file parsing, uuids, small object allocation, etc
  • A Resource Management (RM) framework, including a C client library, and D-Bus-based RM server (written in C++)
  • A number of sample IL components, including an mp3 decoder (based on libmad), a vp8 decoder (based on libvpx), a SDL-based YUV renderer, etc.

{% blocktrans %}Tizonia OpenMAX IL is released under the GNU Lesser General Public License version 3 license.{% endblocktrans %}

{% trans "About Skema, Tizonia's test execution framework" %}

{% blocktrans %}Skema is a Python test tool that allows the definition and execution of arbitrary OpenMAX IL graphs and use cases using an easy-to-program XML syntax. Skema internally uses Python's ctypes to programatically load and use the OpenMAX IL Core. Skema can be easily extended to support new functionality and test scenarios by defining new XML tags also written in Python. {% endblocktrans %}

{% blocktrans %}Skema borrows ideas and some of its code from Linaro's excellent Lava-test framework. {% endblocktrans %}

{% blocktrans %}Skema is released under the GNU General Public License version 3 license.{% endblocktrans %}
